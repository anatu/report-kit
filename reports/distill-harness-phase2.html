The result

On GSM8K, the harness surfaced DeepSeek-R1-Distill-Qwen-1.5B at $0.058/Mtok — clearing the 0.70 quality threshold and beating the previous Qwen2.5-7B-Instruct winner at $0.072/Mtok. We built and proved both tiers end-to-end to get there: a fast tier (model swaps, quantization) and a slow tier (QLoRA distillation). The full cost/quality landscape across all ten candidates:

Takeaway

The cheapest model that clears quality isn't the smallest or the one we trained — it's a pretrained distill someone else already paid to produce. The fast-tier swap found the win the slow tier couldn't.

Findings worth keeping

Pretrained distills beat home-grown distillation

Our QLoRA attempt on Qwen2.5-1.5B (500 examples, rank 32, 3 epochs) reached 0.643 — below even the 0.650 cold baseline, and at higher cost. DeepSeek-R1-Distill-Qwen-1.5B, the same parameter count, hits 0.752 because DeepSeek spent vastly more compute distilling R1 reasoning into it than a 500-example LoRA ever could.

Rule

Before running expensive home-grown distillation, survey HF Hub for task-relevant pretrained distills and add them as candidates first.

AWQ wins are memory-bound, not universal

AWQ-INT4 on Qwen-7B routed to an A10G cost $0.136/Mtok vs $0.072 for BF16 on A100-40GB — the A100 was 3.4× faster, swamping its 1.9× higher hourly rate. Quantization only saves money when memory pressure forces the baseline onto a more expensive GPU class. The cost-optimal pick is not always the smallest GPU that fits.

Eval size materially changes the cost model

At n=200, vLLM startup overhead dominated wall time and inflated $/Mtok by 10–40%, sometimes flipping the ordering. Bumping to n=1000 amortized startup and stabilized costs. Rule of thumb: generation time should dominate container startup + model load — ≥500 examples for a 7B, ≥1000 for smaller models.

LoRA-at-runtime doubles wall time

The same 1.5B base went from 41s to 81s for 1000 examples once the LoRA adapter was applied as a runtime delta — pushing $0.029 to $0.063/Mtok. The fix (peft.merge_and_unload() to fold the adapter in, then serve as a vanilla checkpoint) wasn't implemented; it's the first thing to add.

Limitations & next steps

Methodological improvements, ranked by expected value per unit effort:

  1. Merge LoRA after training (peft.merge_and_unload()). Folding the adapter into the base avoids the runtime-delta penalty, letting a home-grown 1.5B distill compete at $0.029 instead of $0.063/Mtok.
  2. More teacher data — the distillation run used only 500 examples. More data plus completion-only loss and merged inference may push a home-grown 1.5B past the 0.75 mark.
  3. More fast-tier actions: Wanda pruning, prompt compression, speculative-decoder pairing.
  4. A better task than GSM8K — structured extraction, narrow classification, tool-use formatting: no off-the-shelf distills exist and the frontier-API → distilled-1.5B cost gap is largest there.
  5. Latency-aware cost model — adding a latency term would unseat the "thinking" DeepSeek model for real-time use cases.
